================================================================================

Fecha: 2026-05-02
Proposito: Reset completo de la base de datos en dev. Ejecutar en este orden
exacto desde el SQL Editor de Supabase. Cada bloque depende de los anteriores.

NOTAS GENERALES:
  - El paso 1 (teardown) es DESTRUCTIVO. Solo en dev / staging.
  - Despues de cada bloque conviene refrescar el cache de PostgREST
    ("Database > API" en Supabase). Las funciones ya emiten NOTIFY pgrst,
    'reload schema' al final, pero un refresh manual no esta de mas.
  - Si algun script falla a medias, revisar el error, corregir y re-ejecutar.
    Todos los scripts son idempotentes (DROP IF EXISTS + CREATE OR REPLACE).
